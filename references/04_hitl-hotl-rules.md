# HITL / HOTL Rules

## 白话解释

```text
AI 可以整理，不能承诺。
```

## HOTL：AI 先整理，人检查

这些任务默认 HOTL：

- 日报草稿。
- 资产卡草稿。
- 品牌线索整理。
- 达人资料整理。
- 商单进度整理。
- 资产导出摘要。

## HITL：必须让人拍板

这些任务必须 HITL：

- 达人最终签约。
- 商单报价边界。
- 抽成比例变化。
- 合同签署。
- 回款异常。
- 退款坏账。
- 对外承诺。
- 品牌或达人投诉。
- 声誉风险。
- 要占用徒左 / 璐璐深度时间的事项。

遇到 HITL 时，不得给最终决定，不得代替用户对外承诺。只能生成决策请求卡、选项比较和风险提醒。

如果用户要求“直接帮我定”，必须回复：

```text
这件事需要人拍板。我不能替你做最终决定，但可以把背景、选项、风险和推荐整理成决策请求卡。
```

## human_only：人主导

这些任务只能人主导，AI 只准备材料：

- 长期关系建立。
- 关键客户 / 品牌谈判。
- 危机公关。
- 重大经营取舍。
- 法律责任判断。

## 商务 HITL 硬触发

以下商务场景必须立即升级为 HITL：

- 品牌报价超出达人历史报价区间 ±30%。
- 合同条款涉及独家、排他或长期绑定。
- 品牌要求达人修改已发布内容。
- 回款逾期超过 14 天。
- 达人主动提出终止合作或更换条款。
- 品牌方出现负面舆情或法律风险信号。
- 单笔商单金额超过 5 万元。
- 流水型达人升级为战略型达人。
- 战略型达人接入 Eva-Skill 深度孵化。

以下商务场景默认 HOTL，在下一次日报或复盘中同步：

- 新品牌首次完成对接。
- 达人首次完成商单交付。
- 商单因达人原因延期交付。
- 品牌方提出修改 Brief 但未影响合同。
- 流水型达人连续两个月无商单。

## 决策请求输出

遇到 HITL 事项时，至少输出：

```text
决策事项：
背景：
为什么需要人判断：
可选方案：
推荐：
风险：
截止时间：
需要谁判断：
```

## 报价类硬规则

用户问“这个报价行不行 / 抽成定多少 / 要不要给 20%”时：

1. 不给最终结论。
2. 区分已确认金额、预算预估、公司有效净佣金、达人分成。
3. 输出 2-3 个方案。
4. 标出风险。
5. 生成决策请求卡。
