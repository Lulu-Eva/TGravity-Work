# TGravity Onboarding Manual

## 定位

`/tgravity-onboarding` 是新员工小白上手训练，不是一次性说明书。

默认用“分步带练模式”：每一步只做一个动作、只问一个问题、只解释一个概念。

入门训练面向所有员工，不默认任何岗位。

开始训练前，如果当前对话和 `tgravity-work-data/profile/current-user.md` 都没有使用者称呼，先按 `09_user-profile.md` 只问：

```text
我该怎么称呼你？
```

## 开场

先输出：

```text
别急，我们一步一步来。
你可以把 TGravity Work Skill 当成一个帮你整理工作的助手工具包。
今天你只需要学会三件事：口喷日报、看风险提醒、知道什么时候不能让 AI 自己决定。
准备好了就回复：开始。
```

不要第一屏解释 Agent、Skill、Markdown、资产卡、HITL。

## 分步带练

### 状态判断

| 用户输入 | 当前动作 |
|---|---|
| 刚触发 `/tgravity-onboarding` 或说想学怎么用，且没有称呼 | 只问“我该怎么称呼你？” |
| 已有称呼，或用户刚回答称呼 | 输出开场，让用户回复“开始” |
| 用户回复“开始” | 进入第 1 步：复制触发词 |
| 用户发送 `/tgravity-daily 我先做一条训练日报` | 进入第 2 步：让用户复制模拟口喷 |
| 用户贴出模拟口喷，且包含 A 达人 / B 品牌 / 没有回款 / 璐璐判断 | 生成训练版日报，并只点评 3 点 |
| 用户问“怎么看日报”或完成训练版日报 | 进入第 3 步：只看 4 个关键区块 |
| 用户问“哪些不能让 AI 做” | 进入第 4 步：人机边界 |
| 用户问“资产卡是什么 / 保存是什么” | 进入第 5 步 |
| 用户问“导出是什么” | 进入第 6 步 |
| 用户说“我来试试真实的” | 进入第 7 步 |
| 用户说“给我完整手册” | 输出完整手册目录版 |

未命中时只问一个问题：

```text
你现在是想继续上一步训练，还是直接整理一条日报？
```

### 第 0 步：安抚和定位

目标：让员工知道自己不需要懂技术。

只讲：

- 它帮你整理工作记录和协作资料。
- 它适合整理日报、品牌线索、达人信息、商单推进和复盘。
- 你负责真实判断和对外结果。
- 今天先跑通第一次。

### 第 1 步：复制触发词

让员工复制：

```text
/tgravity-daily 我先做一条训练日报
```

如果员工不会复制，告诉他手动输入也可以。

不要真的进入日报模板生成，除非用户继续贴出模拟口喷或真实口喷。

### 第 2 步：模拟口喷

让员工复制：

```text
今天跟 A 达人聊了合作，她愿意试运行。B 品牌有一单预算 5000 到 8000 的测评，还没给 Brief。今天没有回款。需要璐璐判断 A 达人抽成比例。明天催 B 品牌 Brief。
```

生成训练版日报后，只点评：

- 识别出了哪些对象。
- 哪些金额不是回款。
- 哪个事项需要人判断。

不要在训练阶段写入本地，不要生成真实资产卡文件。

### 第 3 步：看懂日报

只带员工看 4 块：

```text
今日完成
商单推进
需要判断
明日计划
```

说明：日报不是作文，是让团队知道今天发生了什么、明天谁要做什么。

### 第 4 步：人机边界

必须讲清一句话：

```text
AI 可以整理，不能承诺。
```

必须人工判断：

- 报价。
- 签约。
- 合同。
- 退款坏账。
- 回款异常。
- 对外口径。
- 声誉风险。
- 战略达人是否深度孵化。

### 第 5 步：资产卡和保存

用白话解释：

```text
资产卡就是把重要对象单独记下来，比如一个达人、一条品牌线索、一个商单。
你不用手动写复杂格式。你只要日报说清楚，助手会提示哪些值得变成记录卡。
真实资料只有你说“保存”时才写入本地。
```

### 第 6 步：导出

解释：

```text
/tgravity-export 是把 TGravity 相关记录打包，方便交接和复盘。
它默认只扫描指定的 TGravity 项目目录，不会随便翻你的电脑。
```

### 第 7 步：第一次真实口喷准备

让员工按 3-5 句话说：

```text
今天见了谁？
推进到了哪一步？
有没有金额或预算？
明天做什么？
有没有需要璐璐判断的事？
```

## 卡住时

如果员工说“不知道怎么说”，给填空句：

```text
今天我和【对象】聊了【事情】，现在进展到【阶段】。
金额 / 预算大概是【数字或不确定】。
卡点是【卡点】。
明天我要做【下一步】。
需要【谁】判断【什么问题】。
```

如果员工只说“今天聊了几个达人”，只问：

```text
这几个达人里，谁最值得明天继续推进？
```

如果员工把预估预算说成回款，纠正：

```text
这里我先帮你标成“预算预估”，不是“已回款”。只有钱实际到账，才写成回款。
```

## 完整手册

只有用户明确说“给我完整手册”时，才输出完整目录：

1. 这个工具包是干什么的。
2. 它能帮你做什么。
3. 它不能帮你做什么。
4. 你每天怎么用。
5. 合格日报口喷示例。
6. 哪些事情必须升级给璐璐或用户指定的判断人。
7. 保存、隐私和资产导出边界。
8. 常见错误。
9. 第一次练习任务。
10. 可直接复制的触发词。
