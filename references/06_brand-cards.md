# Brand Cards

## 目标

记录词元引力对接过的品牌方信息，包括品牌画像、对接状态、适配达人和历史合作记录。

每个有真实接触的品牌线索都应建立品牌卡；未确认真实需求的品牌只输出草稿，不保存。

## 触发

品牌线索、品牌卡、品牌状态更新、品牌适配达人、品牌查询、品牌跟进。

## YAML 头部

复制品牌模板生成真实资产卡时，填完必填字段后设置 `tgravity_asset: true`：

```yaml
---
tgravity_asset: true
asset_id: TGRAVITY-YYYYMMDD-BRAND-001
asset_type: brand_lead
title:
owner:
business_line: 词元引力
status: active
risk_level: low
hitl_level: HOTL
created_at:
updated_at:
source:
tags:
---
```

## 必填字段

```text
品牌名称：
所属行业 / 品类：
品牌规模：大品牌 / 中小品牌 / 新锐品牌
线索来源：主动开发 / 转介绍 / 品牌主动联系
首次接触时间：
当前状态：
当前卡点：
下一步动作：
预计推进时间节点：
维护人：
```

## 对接人信息

涉及姓名、职位、联系方式时，只在用户明确提供后记录。不要编造联系方式，不要从外部猜测个人信息。

```text
主要对接人姓名：
职位：
联系方式：
```

## 商务画像

```text
常规投放预算区间：
偏好内容形式：图文 / 视频 / 直播
偏好平台：小红书 / 抖音 / B站 / 其他
偏好达人类型：垂类 / 泛娱乐 / 知识 / 生活方式
历史合作过的达人类型和量级：
对内容的管控程度：严格 / 适中 / 宽松
回款周期：预付 / 验收后 N 天 / 其他
```

## 管理规则

- 每次品牌沟通后，当天更新状态和下一步动作。
- 超过 30 天没有推进的线索，标记为“暂停跟进”，并记录原因。
- 已结案合作，在结案后 7 天内补全字段并归档。
- 品牌卡不对外共享，仅供内部使用。

## HITL 触发

品牌相关报价、合同、修改已发布内容、负面舆情、法律风险、超常规对外承诺，统一按 `04_hitl-hotl-rules.md` 升级为决策请求卡。
