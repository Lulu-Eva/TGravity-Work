# Save Boundaries

## 总原则

默认不保存真实业务资料。只有用户明确说“保存”“写入本地”“生成并保存资产卡”时，才进入保存流程。

保存流程必须分两步：

```text
1. 先说明默认保存目录。
2. 用户回复“确认保存”后才写入。
```

## 可以保存

- 日报草稿。
- 达人画像卡。
- 品牌线索卡。
- 商单推进卡。
- 商单复盘卡。
- 决策请求卡。
- 异常升级卡。
- 用户明确授权保存的工作资料。

## 不默认保存

- 身份证、银行卡、社保、公章照片。
- 私人聊天。
- 未授权客户隐私。
- 未授权员工隐私。
- 未确认的公司对外口径。
- 与 TGravity 无关的个人资料。

## 保存位置

真实业务资产保存到运行项目：

```text
tgravity-work-data/
```

不要保存到 Skill 仓库本体。

如果当前项目目录不明确，先问：

```text
你要保存到哪个项目目录？如果是当前词元引力项目，我会使用 `tgravity-work-data/`。
```

## 保存前提醒

如果内容包含敏感信息，保存前提醒：

```text
这段内容可能包含敏感信息。我可以先生成草稿，不写入本地；如果你确认要保存，请明确说“确认保存”。
```
