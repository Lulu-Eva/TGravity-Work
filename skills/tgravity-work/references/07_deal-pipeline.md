# Deal Pipeline

## 目标

追踪每一笔商单从线索到回款的完整推进状态，避免中间环节丢失、停滞或无人负责。

## 触发

商单状态、商单推进、推进卡点、回款追踪、报价中、合同推进、验收发布、已结案。

## 商单状态

```text
S0 线索确认：品牌有明确投放需求，已确认预算区间和内容方向。
S1 达人匹配：已向品牌推荐达人方案，等待确认或反馈。
S2 报价中：已提交报价，等待品牌确认或还价。
S3 合同推进：报价已确认，合同起草或审核中。
S4 执行中：合同已签署，达人正在创作或已提交内容。
S5 验收与发布：内容已提交或已发布，等待品牌验收。
S6 回款中：验收已通过，等待品牌打款。
S7 已结案：款项已到账，商单完成，进入复盘。
S_暂停：推进中出现卡点，暂时搁置。
S_终止：商单因任何原因终止，进入复盘。
```

## 状态停留时间

```text
S0 -> S1：不超过 3 个工作日
S1 -> S2：不超过 5 个工作日
S2 -> S3：不超过 7 个工作日
S3 -> S4：不超过 10 个工作日
S4 -> S5：按合同约定交付时间
S5 -> S6：不超过 3 个工作日
S6 -> S7：按合同约定回款周期 + 7 天预警
```

任何状态超时未推进，标记为“卡点”，并在日报中说明原因和处理方案。

## 回款规则

- 回款状态必须区分：未到付款节点 / 已到节点未回款 / 部分回款 / 已回款。
- 回款异常是否触发 HITL，统一读取 `04_hitl-hotl-rules.md`；本文件不维护第二份阈值清单。
- 不要把预算、GMV、报价、佣金、已回款混为一类。

## 推进输出格式

用户问商单推进时，优先输出：

```text
当前状态：
已停留时间：
是否超时：
当前卡点：
下一步动作：
负责人：
是否需要 HITL：
```

## 状态更新规则

- 没有明确证据时，不得把状态推进到下一阶段。
- 用户只说“应该快签了”，只能记录为“合同推进：待确认”，不能写“合同已签署”。
- 进入 S7 或 S_终止 时，提醒生成商单复盘，读取 `08_deal-review.md`。
