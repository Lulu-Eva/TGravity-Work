# TGravity Learning Manual

## 定位

`/tgravity-onboarding` 是 TGravity Work 的通用新手学习框架，不是固定讲稿。

它面向所有员工，不默认岗位，不默认称呼。目标是让员工形成三个习惯：

1. 开工前先问 AI 做任务定义和人机分工。
2. 把口喷、流水账、材料交给 AI 整理成结构化文档。
3. 知道哪些事必须由人判断。

## 总流程

1. 称呼初始化：没有称呼时先读 `09_user-profile.md`，只问“我该怎么称呼你？”。
2. 进度判断：用户说“继续下一课”时，先按“继续规则”恢复。
3. 学习诊断：不知道学哪块时，只问一个选择题。
4. 单步带练：每轮只教一个动作，并让员工完成一次。
5. 反馈调整：根据“会了 / 没懂 / 想实战”决定下一步。
6. 保存进度：只有用户明确要求并确认后，才写入本地。

训练阶段不保存真实业务资料，不生成真实资产卡，不对外承诺。

## 学习诊断

首次进入或用户说“开启新手教程”时，不要展开完整手册。只问：

```text
你现在最想先学哪一块？
1. 我完全不会用，带我从第一步开始。
2. 我想学开工前怎么让 AI 分工。
3. 我想学怎么生成日报。
4. 我想学怎么用搜索技能。
5. 我想学怎么整理视频素材和切片表。
6. 我想知道哪些事不能交给 AI。
```

如果用户说“不知道”，默认从第 1 项开始。

## 继续规则

用户说“继续下一课”时，按顺序处理：

1. 当前对话有未完成模块：继续当前模块。
2. 当前对话无进度，但用户已授权保存过学习进度：读取 `tgravity-work-data/learning/current-user.md`。
3. 如果进度文件存在：按 `next_step` 继续；没有 `next_step` 时，进入第一个未完成模块。
4. 如果没有进度文件：只问“你上次学到哪一块了？如果不记得，我可以从工作分配技能继续。”

学习进度文件只允许保存这些字段：

```yaml
---
tgravity_learning: true
tgravity_asset: false
learner: ""
current_module:
completed_modules: []
next_step:
updated_at:
---
```

不要保存真实业务输入、客户资料、品牌资料、聊天记录或隐私信息。

## 模块 A：完全新手

目标：建立“先问 AI”的基本习惯。

开场：

```text
先不用学术语。你现在只需要学会一个动作：开始工作前，先问 AI 这件事怎么分工。
```

训练动作：

```text
/tgravity-workcheck 我想整理今天跟进过的品牌，看看哪些要继续推进
```

完成后只讲 3 点：

- 它会先帮你定义工作。
- 它会拆出 AI 做什么、人做什么。
- 登录后台、截图、导出数据、对外承诺通常先由人做。

结束追问：

```text
这一步你是已经会了、还没懂，还是想换一个例子？
```

## 模块 B：工作分配技能

目标：让员工理解这是工作前置动作。

必须讲清：

```text
公司要求：当你决定开展某项工作时，先使用工作分配技能。
它会先定义工作内容，再拆出哪些适合 AI，哪些适合人类。
```

训练动作：

```text
把你今天准备做的一件真实工作告诉我。先不用做，我会先判断它能不能交给 AI，以及你要怎么配合。
```

如果用户给出真实工作，读取 `10_workcheck-task-split.md`，但保持训练口径：只演练，不保存，不对外承诺。

## 模块 C：日报生成

目标：让员工能把口喷或工作流水整理成规范日报。

训练动作：

```text
/tgravity-daily 我先做一条训练日报
```

模拟口喷：

```text
今天跟 A 达人聊了合作，她愿意试运行。B 品牌有一单预算 5000 到 8000 的测评，还没给 Brief。今天没有回款。需要判断 A 达人抽成比例。明天催 B 品牌 Brief。
```

生成训练日报后，只看 4 块：

- 今日完成。
- 商单推进。
- 需要判断。
- 明日计划。

必须说明：

```text
日报不是作文。你可以攒一堆事，用语音转文字全部口喷给它，再让它整理。
```

## 模块 D：搜索技能

目标：让员工知道什么时候用 `tgravity-work-search`，但不要在教程中误触发搜索 Skill。

必须讲清：

- 搜索技能适合公开互联网资料、近期信息核验、竞品资料、品牌/达人公开资料调研。
- 它需要单独配置 Perplexity 和 Tavily 两个 API Key。
- API Key 只存在本机，不写进 GitHub。
- 如果其中一个引擎因为没积分、网络或限流失败，搜索技能会自动使用另一个；如果两个都失败，要如实说明原因，无法判断时只说搜索失败。
- Codex 使用时，需要允许脚本执行和联网；权限受限时，先切到完全访问权限或让管理员配置。
- 飞书、微信、小红书后台、抖音后台这类需要登录的内容，先由人导出或截图，再交给 AI 整理。

教程阶段只展示触发词，不要求员工复制执行：

```text
/tgravity-search-status
搜索技能 帮我查 2026 年小红书 AI 垂类 MCN 的公开竞品资料，列出来源链接。
```

然后问：

```text
你现在只是先了解搜索技能，还是要真实配置并使用搜索？
```

如果用户选择真实配置或真实搜索，结束新手教程，切到 `tgravity-work-search`。如果只是了解，继续下一个模块。

## 模块 E：视频分析技能

目标：让员工知道什么时候使用 `tgravity-work-video-indexer`，但不要在教程中误扫描真实视频。

必须讲清：

- 视频分析技能适合线下活动、访谈、课程、工作坊等大量视频素材整理。
- 它能做视频清单、元数据、逐字稿对齐、关键帧/contact sheet、素材切片表草稿。
- 它不替代最终剪辑判断，不判断素材是否一定能公开发布。
- 肖像授权、版权、客户隐私、发布口径、最终剪辑审美必须由人确认。
- 新电脑主要检查 `ffmpeg` / `ffprobe`，不需要 API Key。

教程阶段只展示触发词，不要求员工复制执行：

```text
/tgravity-video-check
视频分析技能，项目目录是【项目路径】，请先生成素材切片表草稿。
```

然后问：

```text
你现在只是先了解视频分析技能，还是要真实处理一个视频项目？
```

如果用户选择真实处理，结束新手教程，切到 `tgravity-work-video-indexer`。如果只是了解，继续人机边界模块。

## 模块 F：人机边界

目标：让员工知道 AI 可以整理，不能拍板。

必须讲清一句话：

```text
AI 可以整理，不能承诺。
```

必须人工判断：

- 报价。
- 签约。
- 合同。
- 退款坏账。
- 回款异常。
- 对外口径。
- 声誉风险。
- 战略达人是否深度孵化。

## 反馈驱动

每轮结束只问一个轻问题：

```text
这一步你是已经会了、还没懂，还是想直接拿真实工作试一下？
```

| 用户反馈 | 下一步 |
|---|---|
| 已经会了 | 跳到下一个模块 |
| 没懂 | 换例子，只重复当前模块 |
| 想实战 | 用真实输入演练，但不保存 |
| 问具体术语 | 用白话解释术语，然后回到当前模块 |
| 想完整手册 | 输出“完整手册目录”，不展开所有正文 |

## 保存学习进度

默认不保存学习进度。

如果员工明确说“保存学习进度”，先说明保存位置：

```text
我会把学习进度保存到 `tgravity-work-data/learning/current-user.md`。这只用于下次继续新手教程，不会写进 Skill 仓库。确认保存吗？
```

用户确认后才保存。保存内容只包括称呼、当前模块、已完成模块、下一步和更新时间。

## 学习顺序

默认顺序：

1. 完全新手。
2. 工作分配技能。
3. 日报生成。
4. 搜索技能。
5. 视频分析技能。
6. 人机边界。

用户指定模块时，直接进入指定模块，不强迫从头学。

## 卡住时

如果员工说“不知道怎么说”，给填空句：

```text
我现在想做【事情】。
这件事和【对象/项目/平台】有关。
我希望最后得到【文档/表格/话术/名单/决策】。
我不确定哪些该我做，哪些该 AI 做。
```

如果员工只说“我今天干了很多事”，引导到日报：

```text
你可以直接把今天做的事口喷给我。我会先帮你整理成日报，不需要你写得很规范。
```
