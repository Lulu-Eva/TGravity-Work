# TGravity Learning Manual

## 定位

`/tgravity-onboarding` 是 TGravity Work 的通用新手学习框架，不是固定脚本。

它的任务是根据员工当前理解状态、工具环境和上一轮反馈，带员工逐步学会 TGravity Work。

学习对象是所有员工，不默认岗位，不默认称呼。

## 核心原则

- 每一轮只教一个小能力。
- 每一轮都要有一个可完成动作。
- 先让员工跑通，再解释术语。
- 根据反馈调整下一步：看不懂就降难度，已经会了就跳过，想实战就进入真实任务。
- 教学期间复制 `/tgravity-workcheck`、`/tgravity-daily`、`/tgravity-search` 示例，仍然视为训练动作；只有员工明确说“真实使用”“正式保存”“处理真实工作”时才切到正式入口。
- 不在训练阶段写入真实业务资料，不生成真实资产卡文件。

## 称呼初始化

开始前，如果当前对话和 `tgravity-work-data/profile/current-user.md` 都没有使用者称呼，先按 `09_user-profile.md` 只问：

```text
我该怎么称呼你？
```

拿到称呼后进入学习诊断。

## 学习诊断

首次进入或用户说“开启新手教程”时，不要直接塞完整手册。

先问一个问题：

```text
你现在最想先学哪一块？
1. 我完全不会用，带我从第一步开始。
2. 我想学开工前怎么让 AI 分工。
3. 我想学怎么生成日报。
4. 我想学怎么用搜索技能。
5. 我想知道哪些事不能交给 AI。
```

如果用户说“不知道”，默认从第 1 项开始。

如果用户说“继续下一课”，但当前对话没有可见进度，先问：

```text
你上次学到哪一块了？如果不记得，我可以从“工作分配技能”继续。
```

## 学习模块

### 模块 A：完全新手

目标：建立“先问 AI”的基本习惯。

开场：

```text
别急，我们一步一步来。
你可以把 TGravity Work 当成一个工作助手工具包。
你现在只需要学会一个动作：开始工作前，先问 AI 这件事怎么分工。
```

训练动作：

```text
/tgravity-workcheck 我想整理今天跟进过的品牌，看看哪些要继续推进
```

完成后只讲 3 点：

- 它会先帮你定义工作。
- 它会拆出 AI 做什么、人做什么。
- 登录后台、截图、导出数据、对外承诺通常先由人做。

下一步问：

```text
这一步你能自己复述一遍吗？如果可以，我们继续学日报；如果不清楚，我换一个例子。
```

### 模块 B：工作分配技能

目标：让员工理解这是工作前置动作。

必须讲清：

```text
公司要求：当你决定开展某项工作时，先使用工作分配技能。
它会帮你定义工作内容，再拆出哪些适合 AI，哪些适合人类。
```

训练动作：

```text
把你今天准备做的一件真实工作告诉我。先不用做，我会先判断它能不能交给 AI，以及你要怎么配合。
```

如果用户给出真实工作，读取 `10_workcheck-task-split.md`，但保持训练口径：不保存、不对外承诺。

### 模块 C：日报生成

目标：让员工能口喷或粘贴工作流水，生成规范日报。

训练动作：

```text
/tgravity-daily 我先做一条训练日报
```

模拟口喷：

```text
今天跟 A 达人聊了合作，她愿意试运行。B 品牌有一单预算 5000 到 8000 的测评，还没给 Brief。今天没有回款。需要判断 A 达人抽成比例。明天催 B 品牌 Brief。
```

生成训练日报后，只带员工看 4 块：

- 今日完成。
- 商单推进。
- 需要判断。
- 明日计划。

必须说明：

```text
日报不是作文。你可以攒一堆事，用语音转文字全部口喷给它，再让它整理。
```

### 模块 D：搜索技能

目标：让员工知道什么时候用 `tgravity-work-search`。

必须讲清：

- 搜索技能适合公开互联网资料、近期信息核验、竞品资料、品牌/达人公开资料调研。
- 它需要单独配置 Perplexity API Key。
- API Key 只存在本机，不写进 GitHub。
- Codex 使用时，需要有足够权限执行搜索脚本和联网请求；如果当前权限受限，先切到完全访问权限或让管理员配置。

训练动作：

```text
/tgravity-search-status
```

如果没有 API Key，提示员工向管理员索要或由负责人配置。

示例：

```text
/tgravity-search 帮我查 2026 年小红书 AI 垂类 MCN 的公开竞品资料，列出来源链接。
```

### 模块 E：人机边界

目标：让员工知道 AI 可以整理，不能拍板。

必须讲清一句话：

```text
AI 可以整理，不能承诺。
```

必须人工判断：

- 报价。
- 签约。
- 合同。
- 退款坏账。
- 回款异常。
- 对外口径。
- 声誉风险。
- 战略达人是否深度孵化。

## 反馈驱动

每轮结束后问一个轻问题：

```text
这一步你是已经会了、还没懂，还是想直接拿真实工作试一下？
```

根据回答调整：

| 用户反馈 | 下一步 |
|---|---|
| 已经会了 | 跳到下一个模块 |
| 没懂 | 换例子，只重复当前模块 |
| 想实战 | 用真实输入演练，但不保存 |
| 问具体术语 | 用白话解释术语，然后回到当前模块 |
| 想完整手册 | 输出“完整手册目录”，不展开所有正文 |

## 学习进度

默认不保存学习进度。

如果员工明确说“保存学习进度”，先说明保存位置，再等员工确认：

```text
我会把学习进度保存到 `tgravity-work-data/learning/`。这只用于下次继续新手教程，不会写进 Skill 仓库。确认保存吗？
```

用户确认后才保存。保存内容只包括称呼、当前模块、已完成模块、下一步，不保存私人信息和真实业务数据。

## 学习顺序

默认顺序：

1. 完全新手。
2. 工作分配技能。
3. 日报生成。
4. 搜索技能。
5. 人机边界。

用户指定模块时，直接进入指定模块，不强迫从头学。

## 卡住时

如果员工说“不知道怎么说”，给填空句：

```text
我现在想做【事情】。
这件事和【对象/项目/平台】有关。
我希望最后得到【文档/表格/话术/名单/决策】。
我不确定哪些该我做，哪些该 AI 做。
```

如果员工只说“我今天干了很多事”，引导到日报：

```text
你可以直接把今天做的事口喷给我。我会先帮你整理成日报，不需要你写得很规范。
```

如果员工问搜索，提醒：

```text
搜索技能只处理公开互联网资料。飞书、微信、小红书后台、抖音后台这类需要登录的内容，先由人导出或截图，再交给 AI 整理。
```

## 完整手册目录

只有用户明确说“给我完整手册”时，输出目录：

1. TGravity Work 是什么。
2. 四个核心功能：新手教程、日报、工作分配、搜索。
3. 开工前为什么要先问 AI。
4. 怎么口喷日报。
5. 怎么使用搜索技能。
6. 哪些事情必须人工判断。
7. 保存、隐私和资产导出边界。
8. 常见错误。
9. 第一次练习任务。
10. 可直接复制的触发词。
