# TGW Asset Cards

你只负责资产卡草稿、更新和保存边界。

## 总原则

- 真实业务资产保存到运行项目的 `tgw-data/`，不要保存到 Skill 仓库。
- 默认只输出草稿；用户明确说保存并确认后才写入。
- 缺字段不硬补，写入“待补字段”并只问一个最短补齐问题。
- 报价、签约、合同、退款、回款异常、对外承诺必须标记为 HITL，不能替人拍板。

## 路由闸门

只有用户明确要求生成、更新、检查或保存资产卡时，才执行本 Skill。

如果用户只是说“需要璐璐判断”“这里要人判断”“有报价 / 合同 / 回款风险”，但没有要求生成资产卡或决策请求卡，不要把这句话当成资产卡任务。按当前任务类型处理；如果无法判断任务类型，只问：

```text
你是要把这件事生成决策请求卡，还是只在当前任务里标记为需要人工判断？
```

## 资产类型

```text
creator_card
brand_lead
deal_progress
deal_review
decision_request
escalation
```

## 模板

```text
assets/templates/creator-card.md
assets/templates/brand-lead-card.md
assets/templates/deal-progress-card.md
assets/templates/deal-review.md
assets/templates/decision-request-card.md
```

模板必须保持 `tgw_asset: false`。正式保存且字段补齐后，副本才设置为 `tgw_asset: true`。

## 达人画像卡

必填：

```text
达人昵称
来源
平台
内容方向
用户人群
当前状态
商业标签
适合品牌类型
交付风险
维护人
下一步动作
```

分层字段：

```text
达人类型：战略型 / 流水型
管理强度：深度孵化 / 轻量维护
是否接入 Eva-Skill：是 / 否
创始人介入级别：高 / 中 / 低
```

战略型升级、分成变化、深度孵化、接入 Eva-Skill，必须生成决策请求卡。

## 品牌线索卡

必填：

```text
品牌名称
所属行业 / 品类
品牌规模
线索来源
首次接触时间
当前状态
当前卡点
下一步动作
预计推进时间节点
维护人
```

联系方式只记录用户明确提供的信息，不猜。

## 商单推进卡

状态：

```text
S0 线索确认
S1 达人匹配
S2 报价中
S3 合同推进
S4 执行中
S5 验收与发布
S6 回款中
S7 已结案
S_暂停
S_终止
```

没有明确证据时，不得把状态推进到下一阶段。

## 商单复盘卡

触发：

- 商单已结案。
- 商单终止。
- 任意阶段停滞超过 14 天。

复盘核心是更新达人池、品牌池、商单推进规则和可自动化流程，不是写情绪总结。

## 决策请求卡

遇到 HITL 事项时，至少输出：

```text
决策事项：
背景：
为什么需要人判断：
可选方案：
推荐：
风险：
截止时间：
需要谁判断：
```

默认判断人是 `璐璐`，用户指定其他人时以用户为准。

## 保存规则

保存前分两步：

1. 说明默认保存目录。
2. 等用户回复“确认保存”。

保存目录：

```text
tgw-data/
├── creators/
├── brands/
├── deals/
├── reviews/
├── decisions/
└── escalations/
```
