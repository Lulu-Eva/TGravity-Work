# Transcript Formats

## 支持输入

脚本第一版优先支持：

- Getnotes/飞书妙记类 Markdown：包含录音时间、章节、说话人时间戳。
- SRT 字幕：包含相对时间码。
- Whisper JSON：包含 segments、start、end、text。
- Whisper TXT/VTT/SRT 可作为人工参考，但只有结构化格式能稳定对齐。

## Getnotes Markdown

推荐格式特征：

```text
录音时间：2026-06-20 13:56:44 ~ 2026-06-20 15:12:00

[00:13:31](https://getnotes.seek:811) **章节标题**

说话人1 [00:04:12] 这里是逐字稿内容
说话人2 [00:04:18] 这里是另一段内容
```

脚本会读取：

- 录音开始时间。
- 章节开始时间和章节标题。
- 说话人。
- 相对录音时间码。
- 绝对时间。

只有存在录音开始时间时，才能稳定和视频拍摄时间对齐。

## SRT

SRT 示例：

```text
1
00:00:01,000 --> 00:00:04,000
第一句字幕
```

SRT 通常只有相对时间。除非文件名或配置能确认它属于某个视频，否则不要跨视频自动对齐。

## Whisper JSON

Whisper JSON 示例：

```json
{
  "segments": [
    {"start": 1.2, "end": 4.8, "text": "这里是转写文本"}
  ]
}
```

Whisper 片段适合补充“无现成逐字稿”或“音频来源低置信”的视频。不要把低置信 Whisper 文本直接当作精确字幕。

## 对齐逻辑

稳定对齐需要两类时间：

- 视频拍摄开始时间：来自 DJI 文件名、通用时间文件名、视频元数据或文件修改时间。
- 逐字稿绝对时间：来自录音开始时间 + 片段相对时间码。

对齐后输出：

- 素材内时间码。
- 绝对时间。
- 来源逐字稿。
- 说话人。
- 原文。

如果缺少任一时间来源，只能生成“待人工复核”的候选记录。
