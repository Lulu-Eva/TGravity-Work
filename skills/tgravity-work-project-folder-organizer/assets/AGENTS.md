# 项目 Agent 运行规则

> 路径权威索引：见 `SOURCE_OF_TRUTH.md`。

1. 每次处理本目录前，先读 `SOURCE_OF_TRUTH.md`。
2. 路径冲突时，以 `SOURCE_OF_TRUTH.md` 为准；无法判断时先报告，不直接改。
3. 修改目录结构前，先输出迁移清单。
4. 未经用户明确确认，不删除、不移动、不重命名原始文件。
5. 敏感目录不得默认外发、上传、复制到公开项目。
6. 合同、证照、公章、身份证、银行卡、签约扫描件、原始素材默认禁动。
7. 文件用途不明时，放入待确认清单，不猜。
8. 执行整理后，更新 `SOURCE_OF_TRUTH.md`。

## 命名原则

- 目录按长期责任边界编号，不按临时任务顺序编号。
- 战略、方案、判断类文档优先使用 `YYYY-MM-DD_主题.md`。
- 历史路径只能标注为历史路径，不得写成当前权威路径。
