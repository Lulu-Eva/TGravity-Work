# Classification Rules

## 发票种类判断顺序

按以下顺序判断：

1. 文本包含 `铁路电子客票`、`买票请到12306` 或 `中国铁路`：`铁路电子客票`。
2. 文本包含 `旅客运输服务`，或项目名称明确是客运服务费，且不是机票/携程代订：`出租车/网约车发票`。
3. 文本包含 `增值税专用发票`：`增值税专用发票`。
4. 文本包含 `电子发票`、`电子普通发票`、`电 子 发 票`：`电子普通发票`。
5. 无法判断时默认 `电子普通发票`，但在复查报告中标记低置信。

## 项目名称总规则

- 尽量按票面原文填写。
- 不要把所有项目统一改成简短类别。
- 不要只靠文件名判断项目名称。
- 发现项目名称为空、`None`、`nan`、`无费` 等明显异常时，加入复查报告。

## 客运服务费

打车/网约车常见项目名称：

```text
*交通运输服务*客运服务费
```

规则：

- 如果票面原文是 `*交通运输服务*客运服务费`，必须按原文写。
- 不要默认写成 `*运输服务*客运服务费`。
- 只有票面原文本来就是 `*运输服务*客运服务费` 时，才可以写这个值。
- 文件名包含打车、约车、出租、高德打车等关键词只能作为辅助线索，不能覆盖票面项目名称。

## 代订机票和代订附加产品

机票/携程类发票不能因为文件名包含 `出行` 就归为网约车。

常见项目名称：

```text
*生产生活服务*代订机票产品
*生产生活服务*代订附加产品
```

规则：

- 文件名或文本包含 `机票`、`携程`、`华程`、`代订机票`、`代订附加` 时，发票种类保持 `电子普通发票`，不要写成 `出租车/网约车发票`。
- 项目名称出现换行时，要合并成一行，例如 `*生产生活服务*代订机票\n产品` 合并为 `*生产生活服务*代订机票产品`。
- `代订机票产品` 和 `客运服务费` 同时出现时，列入人工确认，不要硬判。

## 住宿

常见项目名称：

```text
*生产生活服务*住宿服务
*生产生活服务*住宿费
*住宿服务*住宿费
```

酒店类销售方但项目名称不含 `住宿` 时，加入复查报告。

## 餐饮

常见项目名称：

```text
*生产生活服务*餐饮服务
*餐饮服务*餐饮服务
*餐饮服务*餐饮
*餐饮服务*餐费
```

餐饮类销售方但项目名称不含 `餐` 或 `餐饮` 时，加入复查报告。

## 顺丰/快递

顺丰类项目名称如果识别成：

```text
*生产生活服务*收派服务无费
```

这是异常。常见修正候选：

```text
*收派服务*收派服务费
```

脚本可以给出修正候选，但应在复查报告标注。
