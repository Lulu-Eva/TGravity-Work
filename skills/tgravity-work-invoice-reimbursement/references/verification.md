# Verification

## 自动去重

去重只看 `发票号码`，不看文件名。

规则：

- 同一发票号码只写入第一条。
- PDF 和 OFD 成对时优先 PDF。
- 重复发票必须写入复查报告，列出发票号码和重复文件路径。

## 跨表查重

如果输出目录中已有其他报销表，例如：

```text
发票填写_2026-06.xlsx
发票填写_2026-06-09报销.xlsx
发票填写_2026-06-19报销.xlsx
```

脚本应读取查重范围内这些文件的发票号码，与本次输出交叉检查。

默认查重范围：

1. 输出目录。
2. 用户提供 Excel 模板时，模板所在目录。
3. 用户通过 `--cross-check-dir` 明确追加的目录。

发现跨表重复时，不自动删除，只在复查报告中列出，因为用户可能在做不同批次补充表。

## 每次生成后必须复查

必须检查：

- 表内重复发票号码。
- 6 列是否有空字段。
- 金额字段是否可解析为数字。
- 写入表格的发票是否为公司抬头。
- 类型和项目名称是否存在明显语义错误。
- 跨表重复。

## 重点语义复查

发现以下情况必须报告：

- 机票/携程/华程被识别为 `出租车/网约车发票`。
- 网约车项目名称不是票面常见 `*交通运输服务*客运服务费`，且没有票面原文依据。
- 铁路票发票种类不是 `铁路电子客票`。
- 铁路票销售方不是 `中国铁路`。
- 铁路票项目名称不以 `铁路运输服务`、`补票` 或 `退票费` 开头。
- 酒店类销售方但项目名称不含 `住宿`。
- 餐饮类销售方但项目名称不含 `餐` 或 `餐饮`。
- 项目名称包含 `无费`、`None`、`nan`。

## 复查报告

复查报告至少包含：

```text
原始 PDF 数量
跳过 OFD 数量
写入条数
跳过重复条数
跳过非公司抬头条数
铁路票数量
表内重复
空字段
金额异常
项目名称异常
跨表重复
低置信公司抬头
跨表查重范围
合计金额
```

## 金额复查边界

第一版只检查金额字段是否能被解析为数字，并汇总合计金额。

不要把这一步表述成税务级或 OCR 级“金额核对”。只有在脚本同时保存票面金额原文、归一化金额和 Excel 写入金额，并做二次比对时，才可以称为“金额逐条核对”。

## 公司抬头复查边界

判断顺序：

1. 如果购买方字段已识别，必须以购买方字段为准。
2. 购买方字段包含目标公司名，允许写入。
3. 购买方字段不包含目标公司名，跳过。
4. 购买方字段缺失但全文出现目标公司名，可以写入，但必须标记为低置信公司抬头并提示人工确认。
